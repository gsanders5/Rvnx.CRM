## 2024-04-17 - Centralize Duplicate User Synchronization logic in Web Layer
**Learning:** Found two identical blocks of user synchronization and claim enrichment logic in `Program.cs` (one in token validation, one in fallback middleware). Refactoring this requires handling the request scoped `IServiceProvider` properly. Also, C# analyzer rules enforce `IDE0011` strictly so `if` statements must use curly braces even if one line.
**Action:** When finding duplicate logic in `Program.cs` dealing with `ClaimsIdentity`, extract it as an extension method on `ClaimsIdentity` that takes an `IServiceProvider` to resolve scoped services, and always use curly braces for single-line `if` statements.
